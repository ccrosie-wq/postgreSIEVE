STEP 3 — SIEVE implementation
──────────────────────────────────────

3a. Add SieveShmemSize()  [freelist.c, new function]
    Returns size of SieveState header plus two int32 arrays of NBuffers+1:
      return sizeof(SieveState) + 2 * (n_buffers + 1) * sizeof(int32);

3b. Add SieveInitialize()  [freelist.c, new function]
    Set process-local pointers into shmem:
      SieveCtl  = (SieveState *) algo_block;
      SieveNext = (int32 *)((char *) algo_block + sizeof(SieveState));
      SievePrev = SieveNext + (NBuffers + 1);
    On !found: init sentinel (NBuffers node points to itself):
      SieveCtl->sieve_hand = NBuffers;
      SieveCtl->sieve_head = NBuffers;
      for (i = 0; i <= NBuffers; i++)
          SieveNext[i] = SievePrev[i] = NBuffers;

3c. Add SieveUnlinkLocked()  [freelist.c, internal helper]
    Must be called under buffer_strategy_lock.  Unlinks node 'id' from the
    doubly-linked list, updates hand/head if they pointed to 'id':
      int32 n = SieveNext[id], p = SievePrev[id];
      SieveNext[p] = n;  SievePrev[n] = p;
      if (SieveCtl->sieve_head == id) SieveCtl->sieve_head = n;
      if (SieveCtl->sieve_hand == id) SieveCtl->sieve_hand = p;
      SieveNext[id] = SievePrev[id] = NBuffers;

3d. Add SieveInsertAtHead()  [freelist.c, notify_insert hook]
    Acquire buffer_strategy_lock.  Link new node between sentinel and old head.
    Update sieve_head.  If queue was empty (hand == NBuffers), reset hand to
    the new node.  Release lock.

3e. Add SieveNotifyHit()  [freelist.c, notify_hit hook]
    Lock-free CAS on BufferDesc.state: if usage_count == 0, set to 1 (visited).
    Uses existing BUF_USAGECOUNT_ONE / BUF_USAGECOUNT_MASK macros.
    No-op if usage_count already > 0 (already visited).

3f. Add SieveRemove()  [freelist.c, notify_invalidate hook]
    Acquire buffer_strategy_lock; call SieveUnlinkLocked(buf->buf_id); release.

3g. Add SieveGetBuffer()  [freelist.c, get_buffer vtable slot]
    Loop under buffer_strategy_lock:
      Read hand position.  If hand == NBuffers: elog(ERROR, queue empty).
      Read buf->state for the buffer under hand.
      If visited (usage_count > 0): clear visited via atomic AND, advance hand
        to SievePrev[hand], release lock, continue.
      If pinned (refcount > 0): advance hand, release lock, continue.
      Otherwise: victim found.  Call SieveUnlinkLocked(hand).  Release lock.
      Try LockBufHdr(victim).  If refcount still 0: return victim.
      Else UnlockBufHdr + SieveInsertAtHead (re-queue) and retry.
    Increment StrategyControl->completePasses when hand wraps past
    sieve_head (full traversal), for bgwriter compatibility.

3h. Populate SieveVtable  [freelist.c]
      static const EvictionVtable SieveVtable = {
          .get_buffer        = SieveGetBuffer,
          .notify_hit        = SieveNotifyHit,
          .notify_insert     = SieveInsertAtHead,
          .notify_invalidate = SieveRemove,
          .shmem_size        = SieveShmemSize,
          .initialize        = SieveInitialize,
      };

4. OPEN QUESTIONS / RISKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

A. StrategyShmemSize() call ordering
   BufferShmemSize() calls StrategyShmemSize() before StrategyInitialize(),
   so ActiveEviction must be set earlier than StrategyInitialize() when the
   vtable dispatch in StrategyShmemSize() is activated (STEP 2f).
   Options: (1) read GUC and set ActiveEviction at top of StrategyShmemSize()
   if it is still NULL; (2) introduce a lightweight StrategyEarlyInit() called
   from PostmasterMain before BufferShmemSize().

B. Lock ordering: buffer header lock vs. buffer_strategy_lock
   SieveRemove() is called from InvalidateBuffer() while holding the buffer
   header spinlock.  SieveRemove() then acquires buffer_strategy_lock.
   Ensure no code path acquires these in the opposite order (strategy lock
   then header lock) to avoid deadlock.  Current ClockSweep code never holds
   strategy lock while acquiring a header lock, so this ordering is safe.

C. completePasses semantics in SIEVE
   bgwriter uses completePasses to estimate write pressure.  ClockSweep
   increments it on every full circular scan of NBuffers slots.  SIEVE should
   increment it every time the hand completes a full traversal of the queue
   (returns to sieve_head from the tail direction).  Implement in
   SieveGetBuffer() with a traversal counter.

D. StrategySyncStart() with SIEVE
   StrategySyncStart() currently computes the sync start buffer as
   nextVictimBuffer % NBuffers, relying on ClockSweep's monotonically
   increasing counter.  Under SIEVE, the equivalent is sieve_hand.  A SIEVE
   implementation of this function should return SieveCtl->sieve_hand
   when EVICTION_SIEVE is active.  Can be handled with a simple if/switch on
   StrategyControl->active_policy inside StrategySyncStart().
